Structure and Basic Styles

➤ CSS Syntax

CSS (Cascading Style Sheets) is used to style and visually format HTML elements. It controls how content appears: colors, fonts, spacing, layout, etc.

Explanation of CSS Syntax:

CSS syntax is made up of selectors and declaration blocks. A declaration block contains one or more declarations separated by semicolons.

selector {
    property: value;
  }

Example:

Output Preview

Explanation of the Example:

  • h1 is the selector — it targets all <h1> elements.
  • color: blue; changes the text color to blue.
  • font-size: 24px; sets the font size to 24 pixels.

Key Points to Note:

  • Each declaration ends with a semicolon ;.
  • The selector is followed by curly braces { }.
  • Each property-value pair is a rule applied to the selector.

Inline, Internal, External CSS and Selectors

➤ CSS Inclusion Methods

CSS can be applied to HTML in three ways: Inline, Internal, and External. Each method serves different use cases based on the scope and maintainability of styles.

Explanation:

  • Inline CSS: Applied directly within an element using the style attribute. Best for quick, one-off styling.
  • Internal CSS: Written inside a <style> tag in the HTML <head>. Useful for single-page projects.
  • External CSS: Stored in a separate .css file and linked using <link>. Ideal for maintaining large websites.

Example:

Output Preview

Explanation of the Example:

  • style="color: red;" changes the paragraph color using inline CSS.
  • <style> block inside the <head> applies internal CSS to <h1>.
  • <link rel="stylesheet"> is how you include external CSS (assumed to be in a file named styles.css).

Key Points to Note:

  • Use inline CSS only when necessary.
  • Internal CSS is good for small or single-page projects.
  • External CSS is preferred for modularity and reusability.

Inline, Internal, External CSS and Selectors

➤ CSS Selectors

CSS selectors are patterns used to select and style HTML elements. They allow you to target elements by tag name, class, ID, or relationships.

Common Types of Selectors:

  • Universal Selector (*): Targets all elements.
  • Element Selector (p, div): Targets elements by tag name.
  • Class Selector (.classname): Targets elements with a specific class.
  • ID Selector (#idname): Targets an element with a specific ID.
  • Group Selector (h1, p): Targets multiple elements together.

Example:

Output Preview

Explanation of the Example:

  • *: Applies Arial font to all elements.
  • p: Styles all paragraphs with green text.
  • .highlight: Targets paragraphs with class "highlight" to add background color.
  • #main-title: Applies blue color and large font to the element with ID "main-title".
  • h1, h2: Aligns both headings to the center.

Key Points to Note:

  • # targets by ID (unique), . targets by class (reusable).
  • Use meaningful and unique IDs and classes for maintainability.
  • Selectors let you apply styles efficiently without repeating code.

➤ Box Model Basics

Every HTML element is a rectangular box consisting of content, padding, border, and margin. This is known as the Box Model.

Structure:

        Content → Padding → Border → Margin

Box Model Properties:

  • content: The actual text/image or content.
  • padding: Space between content and border.
  • border: The boundary around the padding.
  • margin: Space outside the element’s border.

Output Preview

Explanation of the Example:

  • box-sizing: border-box; helps keep width predictable by including border/padding in width.

Box Model, Height and Width

➤ Box Model Details

The CSS Box Model is the foundation of layout design. It treats each HTML element as a rectangular box made up of 4 layers:

  • Content: The actual text or image inside the box.
  • Padding: Clears space around the content inside the box.
  • Border: Wraps around the padding (if any) and content.
  • Margin: Clears space outside the border, separating elements.

Properties:

  • width: Horizontal size of the content box.
  • height: Vertical size of the content box.
  • min-width / max-width: Set boundary limits.
  • min-height / max-height: Restrict height range.

Display and Position

➤ Display Properties

The display property determines how an HTML element is rendered in the document flow.
It controls whether the element behaves like a block, inline, or other layout types.

Common Values:

  • block: Takes full width; starts on a new line (e.g., <div>, <p>).
  • inline: Takes only the required width; sits within line (e.g., <span>, <a>).
  • inline-block: Like inline but supports width/height.
  • none: Hides the element completely (no space).

Example:

Output Preview

Explanation:

  • block takes the full line and forces the next element down.
  • inline allows elements to sit side by side, without block-level spacing.
  • inline-block behaves inline but lets you set width/height.

Key Points to Note:

  • display: none removes an element from layout and screen.
  • Use inline-block for button rows or aligned boxes with fixed sizes.

➤ Positioning Elements

The position property controls how an element is placed in the document.
Combined with top, left, right, and bottom properties.

Types of Position:

  • static: Default flow (no offset).
  • relative: Offset from its normal position.
  • absolute: Positioned relative to nearest non-static ancestor.
  • fixed: Positioned relative to the viewport (always stays in place).
  • sticky: Acts like relative until scroll, then sticks.

Example:

Output Preview

Explanation:

  • relative moves from its normal spot, keeping its space.
  • absolute is removed from flow and placed based on parent.
  • fixed stays visible even when scrolling.

Key Points to Note:

  • Use relative to adjust position while keeping flow.
  • absolute is useful for dropdowns, modals, or overlays.
  • fixed is perfect for sticky navbars or "back to top" buttons.

Flexbox and Grid Layout

➤ Flexbox Introduction

Flexbox is a one-dimensional layout method used to align and distribute space among items inside a container. It works in rows or columns, making it perfect for responsive layouts.

Flex Container Properties:

  • display: flex – Activates flex mode for the container.
  • flex-direction – Direction of items: row, column, etc.
  • justify-content – Horizontal alignment (main axis).
  • align-items – Vertical alignment (cross axis).
  • gap – Space between flex items.

Example:

Output Preview

Explanation:

  • display: flex activates Flexbox layout for the container.
  • justify-content: space-around evenly distributes space between items.
  • align-items: center vertically centers them inside the container.

Key Points to Note:

  • Flexbox is best for one-dimensional layouts (row or column).
  • Great for navigation bars, item alignment, and simple layouts.
  • Flexible and responsive by default.

➤ CSS Grid Introduction

CSS Grid is a two-dimensional layout system. It allows you to design layouts using rows and columns. It's ideal for complete page layouts and dashboard-type structures.

Grid Container Properties:

  • display: grid – Activates Grid layout.
  • grid-template-columns – Defines number and size of columns.
  • grid-template-rows – Defines row sizes.
  • gap – Adds spacing between rows and columns.

Example:

Output Preview

Explanation:

  • grid-template-columns: repeat(3, 1fr) creates 3 equal-width columns.
  • gap: 15px adds space between the grid items.

Key Points to Note:

  • CSS Grid is two-dimensional – controls both rows and columns.
  • Best for overall layout structures like websites and dashboards.
  • Supports advanced placement and alignment with minimal code.

➤ Flexbox vs Grid: Comparison

Feature Flexbox Grid
Layout Type One-dimensional (row OR column) Two-dimensional (row AND column)
Best Use Case Navbars, cards in a row, simple alignment Complex layouts, dashboards, full page designs
Control Over Items Item-level control Container-level grid layout control
Learning Curve Easy to learn and apply More advanced concepts (areas, spans)

Summary:

  • Use Flexbox when you only need layout in one direction (horizontal or vertical).
  • Use Grid when working with rows and columns simultaneously.
  • You can combine both in complex layouts (e.g., Grid outer structure, Flex inside items).

Transforms and Animations

➤ CSS Transforms

CSS Transforms allow elements to be visually manipulated using functions such as rotate, scale, translate, and skew. It doesn’t affect document flow—only appearance.

Common Transform Functions:

  • translate(x, y) – Moves element horizontally/vertically.
  • rotate(angle) – Rotates element by degrees.
  • scale(x, y) – Enlarges or shrinks the element.
  • skew(x, y) – Slants the element along axes.

Example:

Output Preview

Explanation:

  • transform: rotate(15deg) rotates the box by 15 degrees clockwise.
  • The element appears rotated, but still occupies the same original space in layout.

Key Points:

  • Transforms are useful for effects like flipping cards, zooming, or rotating items.
  • They can be combined: transform: rotate(15deg) scale(1.2);

➤ CSS Transitions

Transitions allow you to smoothly animate property changes (like color, size, etc.) over a specific duration.

Key Properties:

  • transition-property – Which property to animate.
  • transition-duration – How long the transition takes.
  • transition-timing-function – Easing function (e.g., ease-in, linear).
  • transition-delay – Wait time before starting transition.

➤ CSS Animations

CSS Animations define keyframes that describe how elements change at various points in time. They can loop and run automatically.

Main Properties:

  • @keyframes – Defines the animation steps.
  • animation-name – Binds the keyframes to element.
  • animation-duration – Time the animation takes.
  • animation-iteration-count – Number of loops.
  • animation-timing-function – Speed curve (ease, linear, etc.).

Responsive Design ensures your website looks good on all devices—desktops, tablets, and mobile phones—by adapting layout and styles based on screen size.

Why Responsive Design Matters:

  • Improves user experience across devices.
  • Reduces need for separate mobile websites.
  • Important for SEO and accessibility.

➤ Media Query Usage

Media Queries apply different CSS rules based on the characteristics of the user’s device (screen width, orientation, resolution).

Syntax:

  /* CSS Rules */
}
